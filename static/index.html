Services Registry

Register Service

POST to "/services"

Required Data in Body

  • name : Service Name
  • category : Service Category
  • version : Service Version
  • description : Description of Service
  • endpoint : URL to the Service
  • heartbeatIntervalms : Expected Heart Beat Interval in Milliseconds
  • documentationURL : URL to the Service Documentation

A Successful Registry will Return {id: Unique Id} The required data and the id Should be provided with the heartbeat Request. Registry will purge services with no heartbeat request for longer than 1.5 times heartbeat interval.

Each Service will be Given and identity made of name category and version. Identity needs to be unique.

Search for Service

GET to "/services"

Search Services with Query Parameters

To Request a Specific Service Use the identity key: ?identity=service-identity

Array of Services is Returned

Request Response

  {
    status: 'Success' or 'Failed'
    data: data or null for error,
    error: error message or null
  }
